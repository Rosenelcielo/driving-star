# Driver Star

Driver Star is a Next.js web game that turns an intelligent-cockpit preference test into a playable journey. The project is ready to be deployed as a public website instead of being used only through a local PowerShell session.

## Tech Stack

- Next.js App Router
- React
- TypeScript
- Browser `localStorage` for local progress and report history
- Server-side OpenAI call for the AI-enhanced result narrative

## Local Development

1. Install dependencies:

```bash
npm install
```

2. Create a local env file from the example:

```bash
copy .env.example .env.local
```

3. Fill in:

- `OPENAI_API_KEY`
- `OPENAI_REPORT_MODEL` (optional, defaults to `gpt-4.1-mini`)

4. Start the dev server:

```bash
npm run dev
```

5. Open [http://localhost:3000](http://localhost:3000)

## Production Build

```bash
npm run build
npm run start
```

## Deploy To Vercel

1. Push this repository to GitHub, GitLab, or Bitbucket.
2. Import the repository into Vercel.
3. Add these environment variables in the Vercel project settings:
   - `OPENAI_API_KEY`
   - `OPENAI_REPORT_MODEL`
4. Trigger a deployment.
5. Open the generated public URL.

## Runtime Notes

- The game UI and routes are served as a standard Next.js web app.
- The AI report is generated by `POST /api/report` on the server side.
- If `OPENAI_API_KEY` is missing or the OpenAI request fails, the result page falls back to the built-in rule-based copy.
- User progress and report history remain in the browser via `localStorage`, so no database is required for the current version.
