# 智能座舱卡牌测试小游戏 V2 产品定义稿

## Summary

这是一个网页在线的轻策略卡牌测试游戏。用户通过一段带有星际旅程感的选择过程，在连续遭遇事件、打出功能卡、使用技能卡的过程中，逐步暴露自己真正偏好的智能座舱功能与体验，并最终获得一份兼具游戏感、人格洞察与功能推荐的结果报告。

产品的核心不是传统问卷式“填写答案”，也不是以输赢为目标的强对抗游戏，而是通过 `事件卡 -> 功能卡 -> 技能卡` 的互动结构，把用户对智能座舱的真实需求转化为一段可玩、可看、可解释的旅途。

主流程已确定为：

`首页 -> 开始旅途 -> 旅途设定页 -> 游戏中 -> 结果报告 -> 模式切换`

其中：
- `结果报告` 负责第一时间做情绪化、游戏化的结果揭晓
- `模式切换` 负责将同一份结果切换为 `用户视角` 与 `销售视角` 两种解读方式
- 首页不是三入口并列结构，而是 `开始旅途` 为主入口，`查看报告 / 浏览卡牌库` 为次入口

## Product Definition

### 1. 世界观与旅程结构

用户的旅程从一个 `出生星球` 开始。出生星球不是能力起点，而是审美起点，只影响整局视觉氛围与最终结果包装方式。

旅途结构如下：
- 用户从出生星球出发
- 依次跳跃经过 `5 个主星球`
- 每个主星球承载 `1 张事件卡`
- 每张事件卡包含 `2 步选择`
- 旅途中可插入 `1 到 3 颗小星球`
- 最终抵达第 `6 个星球`
- 第 6 个星球即用户专属的 `驾驶星球`，也是结果揭晓的核心意象

这条旅途本质上是一个偏好识别过程：
- 主星球负责叙事主流程
- 小星球负责挑战与荣誉感
- 终点星球负责结果归纳与身份命名

### 2. 首页与入口结构

首页的目标是让用户一眼理解这不是普通问卷，而是一场可玩的座舱偏好测试旅程。

首页入口层级：
- `主入口`：开始旅途
- `次入口`：查看报告、浏览卡牌库

说明：
- 普通用户默认被引导进入游戏主流程
- 已经完成测试的用户可从次入口回看结果
- 对卡牌体系有强兴趣的研究、维护、品牌产品人员可从次入口进入卡牌库
- 不设置前置的模式选择页作为主流程必经节点

### 3. 旅途设定页

旅途设定页在开始游戏前出现，承担两项配置：

1. `出生星球选择`
- 共 5 个出生星球选项
- 以模糊色系、不同外观气质进行展示
- 不影响六维分值、事件池、功能卡倾向或人格计算
- 仅影响：
  - 整局视觉主题
  - 结果报告配色
  - 车型推荐中的内外饰颜色建议

2. `局长短与小星球设置`
- 支持配置旅途中插入 `1 到 3 颗小星球`
- 支持自定义小星球出现在 5 个主星球之间的具体位置
- 用于控制每局节奏长短与挑战密度

### 4. 卡牌类型定义

第一版卡牌分为 3 类：

1. `功能卡`
- 是智能座舱功能与体验的主要承载体
- 是唯一直接影响六维分值的卡牌类型
- 也是结果报告与推荐逻辑的核心数据来源

2. `事件卡`
- 代表与车相关、驾车出行相关的场景问题
- 用于提出情境、制造语境和选择压力
- 不直接影响六维分值

3. `技能卡`
- 代表对当前手牌、候选牌、牌组结构的操作能力
- 用于增强策略感、补救感和卡牌玩法感
- 不直接影响六维分值

### 5. 双层标签体系

采用 `双层体系`，并预留后台配置能力。

#### 第一层：用户需求维度

默认使用 6 个维度：
- 安全
- 便捷
- 娱乐
- 家庭
- 个性
- 智能

规则：
- 功能卡在六维上采用 `-2 / -1 / 0 / 1 / 2` 打分
- 单张功能卡通常影响 `2 到 3 个维度`
- 整体为 `混合型` 体系
- 即允许一张卡在某些维度加分、在另一些维度减分，以制造真实偏好取舍

说明：
- 这 6 个名称当前是默认模板
- 后续支持自定义修改文案或配置，但第一版规则先围绕这套默认模板建立

#### 第二层：产品能力维度

产品能力一级类目固定为 6 类：
- 语音与交互
- 影音娱乐与内容体验
- 座舱氛围与舒适体验
- 空间与家庭关怀
- 设备互联与生态扩展
- 个性化与形象表达

规则：
- 每张功能卡归属 `1 到 2 个一级类目`
- 每张功能卡关联 `若干二级功能点`
- 用户视角主要引用第一层
- 销售视角主要引用第二层

### 6. 事件回合逻辑

每个主星球包含一张事件卡，每张事件卡按照两步完成：

1. `选择应对方向`
- 用户先对当前场景做方向性选择
- 这个选择不是直接得分，而是为下一步的功能卡选择建立语境

2. `打出功能卡`
- 系统会高亮当前牌组中可响应该方向的功能卡
- 但高亮只是建议，不是限制
- 用户仍可以自由打出未高亮的功能卡

补牌规则：
- 若用户觉得当前没有合适的牌，可主动点击补牌按钮
- 系统抽出 `5 张候选牌`
- 用户从中选择 `1 张`
- 该牌加入牌组，并同时视为本次打出

这个机制的核心原则是：
`系统给建议，但不替用户做决定`

### 7. 技能卡规则

技能卡仅在 `打出功能卡之前` 使用。

使用窗口：
- 普通事件中：每次遇见事件卡时可依次使用技能卡
- 对战场景中：每次最多可使用 `3 次技能卡`

第一版技能卡功能固定为 4 类：
- `重抽`
- `补牌`
- `换牌`
- `复制牌`

语义约定：
- `重抽`：刷新当前抽牌或候选牌结果
- `补牌`：额外获得新的候选或手牌机会
- `换牌`：替换当前不理想的牌
- `复制牌`：复制某张已有功能卡供本次或后续使用

第一版不再增加更多技能牌类型，优先保证规则清晰。

### 8. 小星球挑战机制

旅途中可插入 `1 到 3 颗小星球`，作为主流程中的挑战关卡。

规则：
- 小星球会检验用户当前的 `六个维度`
- 只要某个维度达到该星球设定门槛，即获得该维度对应的 `胜利星`
- 用户可在不同小星球获得不同类型的胜利星
- 不要求传统意义上的“通关”或“失败重来”
- 无论是否获得胜利星，旅程都继续推进

胜利星作用：
- 不改变主测试结果
- 作为荣誉标记出现在最终报告的 `用户名称后缀` 中
- 用于增加成就感、差异感和可展示性

## Page Structure And Wireframe

### 1. 首页

目标：
- 建立世界观
- 传达“这不是无聊问卷，而是一场旅途测试”
- 引导普通用户立即开始

建议模块：
- 主标题
- 副标题
- `开始旅途` 主按钮
- `查看报告 / 浏览卡牌库` 次入口
- 极简玩法说明
- 强化星际旅途、跳跃、驾驶星球等视觉暗示

### 2. 旅途设定页

建议模块：
- `出生星球选择区`
- `小星球数量选择区`
- `小星球插入位置设置区`
- 开始旅途确认按钮

视觉原则：
- 让用户感到自己是在“设定这趟旅途”
- 而不是在填写复杂配置项

### 3. 游戏主界面

采用 `上中下三层固定布局`。

#### 上层：状态层
- 六维分值展示
- 当前旅程进度
- 当前主星球/小星球标识
- 已获得胜利星
- 当前局内阶段提示

#### 中层：旅途主舞台
- 星际地图
- 当前星球
- 跳跃路径
- 事件卡展示
- 主角飞船/车辆的前进表现
- 星球切换与抵达动画

#### 下层：操作层
- 当前手牌区
- 技能卡区
- 当前应对方向提示
- 被系统高亮的推荐功能卡
- 主动补牌按钮
- 当前选择确认区

### 4. 结果报告页

结果报告页负责第一时间做 `游戏结算式揭晓`，更偏情绪价值和仪式感，而不是完整专业分析。

建议内容：
- 游戏结算式标题
- 驾驶星球揭晓
- 主人格 + 副人格
- 一段更像人格测试的自然语言结论
- 一句系统播报式功能推荐总结
- 用户名称与胜利星后缀展示
- 继续查看按钮，引导进入模式切换页

说明：
- 这里不是承载所有分析内容的长报告页
- 它更像是“第一时间揭晓结果”的短结算页

### 5. 模式切换页

该页位于结果报告之后，是完整结果的双视图承载层。

结构已确认如下：
- 顶部有 `2 个按键通道`
- 点击进行切换
- 分别为：`用户视角`、`销售视角`

两种视角的关系：
- `同结构不同文案`
- 即共用同一套模块顺序和信息骨架
- 只调整：
  - 文案语气
  - 标签命名
  - 推荐重点
  - 说明方式

共用的 5 个模块为：
1. `总结果`
2. `六维画像`
3. `功能组合`
4. `旅途回放`
5. `延伸建议`

#### 模块定义

1. `总结果`
- 用户视角：更像人格测试和旅途总结
- 销售视角：更像用户偏好概览与推荐结论摘要

2. `六维画像`
- 用户视角：强调“你更在意什么”
- 销售视角：强调“需求重心落在哪里”

3. `功能组合`
- 用户视角：强调“适合你的座舱组合”
- 销售视角：强调“推荐功能包、能力类目、功能点依据”

4. `旅途回放`
- 不是卡牌清单
- 以关键场景和关键选择的方式回看用户旅程
- 用户视角更偏叙事
- 销售视角更偏选择依据解释

5. `延伸建议`
- 用户视角：给出轻量车型方向与颜色建议
- 销售视角：给出更便于导购沟通的延展建议

### 6. 卡牌库页面

卡牌库不是普通用户主流程重点页，而是偏专业使用。

面向对象：
- 系统维护人员
- 品牌产品研究人员
- 对完整卡牌体系极有兴趣的人

建议能力：
- 卡牌列表视图
- 类型筛选
- 标签筛选
- 产品能力类目筛选
- 搜索
- 标记/星标
- 上传卡牌
- 删除卡牌
- 启用/停用
- 卡牌详情查看

## Report And Recommendation Logic

### 1. 人格结论逻辑

第一版采用：
- `主人格`
- `副人格`
- `一段自然语言结论`

计算依据：
- `六个用户需求维度总分` 决定主人格主轮廓
- `功能卡选择倾向` 进一步细化副人格差异
- 事件卡本身不直接作为人格标签的主判断项

人格标签数量：
- 第一版预设 `6 种`
- 风格偏 `专业感`

示例命名方向：
- 智能效率型
- 安全掌控型
- 家庭关怀型
- 沉浸体验型
- 个性表达型
- 便捷顺滑型

文案风格已确定：
- 标题像游戏结算
- 正文像人格测试
- 功能推荐像系统播报

### 2. 功能推荐逻辑

输出结构固定为两层：
1. `功能组合包`
2. `具体功能点展开`

生成规则：
- 根据六维总分识别前 2 到 3 个高偏好方向
- 再结合功能卡所属的产品能力类目与功能点倾向
- 映射出 2 到 3 个最适合的功能组合包
- 再从组合包中展开具体功能点

说明：
- 用户视角更重组合包的理解性与故事感
- 销售视角更重一级能力类目与二级功能点的可推荐性

### 3. 车型推荐逻辑

第一版定位为 `弱推荐`，仅作延伸层，不作为报告核心。

输出原则：
- 不直接绑定具体品牌或车型
- 给出适合的车型方向
- 给出与出生星球配色相呼应的内外饰颜色建议

也就是说，出生星球会回收到结果页里，作为：
- 视觉风格闭环
- 审美偏好闭环
- 颜色建议依据

## Test Plan

### 1. 主流程验证
- 首页可从主入口进入开始旅途
- 首页次入口可进入查看报告和浏览卡牌库
- 主流程严格符合：`首页 -> 开始旅途 -> 旅途设定页 -> 游戏中 -> 结果报告 -> 模式切换`
- 结果报告后可顺利进入模式切换页，而不是返回首页或进入旧式模式菜单

### 2. 旅途设定验证
- 出生星球仅影响视觉主题、报告配色、颜色推荐
- 出生星球不影响六维、事件池、功能卡计算
- 小星球数量与插入位置可配置
- 不同局长短配置可正常生成对应流程

### 3. 游戏逻辑验证
- 每个主星球对应一张事件卡
- 每张事件卡都按“两步选择”推进
- 系统可高亮推荐功能卡，但不限制自由出牌
- 补牌按钮由用户主动触发
- 补牌后可从 5 张候选牌中选 1 张并立即视为打出
- 技能卡仅能在打出功能卡前使用
- 普通事件与对战场景中的技能使用规则正确

### 4. 小星球挑战验证
- 小星球可按配置插入指定位置
- 小星球按六维门槛发放对应胜利星
- 无论是否获得胜利星都不会中断旅途
- 胜利星可正确展示在结果页用户名后缀中

### 5. 报告与双视图验证
- 结果报告页可正确展示游戏结算式结果
- 模式切换页顶部有两个通道按钮并可切换
- 两种视角共用 5 模块结构
- 两种视角结构一致，仅文案与推荐重点不同
- 用户视角与销售视角都能正确呈现总结果、六维画像、功能组合、旅途回放、延伸建议

## Assumptions And Defaults

- 第一版是网页在线体验，不包含登录、云存档、多人分享
- 第一版优先使用本地卡牌数据与规则映射，不使用 AI 实时生成报告
- 六维名称先固定为 `安全、便捷、娱乐、家庭、个性、智能`，但后台需预留自定义能力
- 产品能力一级类目第一版固定为 6 个，二级功能点后续逐步完善
- 小星球的具体数值门槛、胜利星命名体系、以及人格映射表仍属于下一轮细化内容
- 本稿已完成第一轮产品定义，下一轮应继续细化：
  - 卡牌字段结构
  - 人格映射表
  - 功能组合包映射表
  - 页面模块级线框
  - 关键交互动效说明
